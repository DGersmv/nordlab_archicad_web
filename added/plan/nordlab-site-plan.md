# NORDLAB — сайт-каталог плагинов для Archicad

План проекта для Cursor. Контент (тексты, видео, скриншоты, характеристики плагинов) вставляется отдельно — все места помечены `[CONTENT]`.

---

## 1. Цель и аудитория

- **Работа сайта:** за 30 секунд убедить посетителя, что инструменты реальные и работают, и довести до контакта/покупки.
- **Аудитория:** (а) архитекторы-пользователи Archicad, (б) производители стройпродукции (GDL-библиотеки), (в) бюро, заказывающие кастомную разработку.
- **Источники трафика:** Graphisoft Community forum, Graphisoft Job Board, Telegram-канал API_Archicad, прямые ссылки. Каждая страница плагина = самостоятельный лендинг для прямой ссылки.
- **Анти-цель:** не арт-портфолио, не scroll-experience. Информация быстрее эффектов.

## 2. Стек

- Next.js 14+ (App Router), статическая генерация (SSG), деплой на Vercel
- TypeScript, Tailwind CSS (токены через CSS custom properties)
- Анимации: CSS + минимальный GSAP только там, где указано (раздел 7). Без Lenis, без scroll-hijack
- three.js — только для signature-элемента на главной (lazy, dynamic import)
- next-intl (или аналог) для EN/RU
- Контент плагинов: MDX или typed TS-объекты в `/content` — без CMS

## 3. Структура роутов

```
/                      Главная: hero + каталог плагинов + кто я + контакт
/plugins/[slug]        Страница плагина (единый шаблон)
/for-manufacturers     GDL-библиотеки для производителей (отдельный язык выгоды)
/custom                Кастомная разработка под заказ
/about                 Кратко: Registered API Developer, практикующий архитектор
/changelog             Общий живой changelog всех плагинов
```

EN — корень, RU — `/ru/...`. Переключатель в шапке.

## 4. Дизайн-система

### Палитра (светлая тема, единственная)

```css
--paper:    #FCFBF7;  /* фон, тёплый бумажный белый */
--ink:      #1B1D1F;  /* основной текст */
--graphite: #6B7075;  /* вторичный текст, подписи */
--hairline: #D9D7D0;  /* линии, рамки, разделители */
--pen:      #2547C8;  /* единственный акцент: ссылки, CTA, активные состояния */
--marker:   #18A06A;  /* только для статусов "совместимо/работает" — экономно */
```

Правило: `--pen` — один акцент на экран. Никаких градиентов, теней глубже `0 1px 2px`.

### Типографика

- **Display + body:** Schibsted Grotesk (Google Fonts) — заголовки 600, текст 400
- **Mono:** IBM Plex Mono — версии AC, параметры, changelog, таблицы совместимости, цифры
- Шкала: 14 / 16 / 20 / 28 / 44 / 64. Заголовки с letter-spacing −0.02em
- Всё «машинное» (версии, даты, данные) — только моно. Это системообразующий приём

### Графический язык: культура чертежа

- Разделители секций — **размерная линия**: тонкая линия с засечками-стрелками на концах и подписью моно по центру (компонент `<DimensionRule label="..." />`)
- Иллюстрации — линейная графика 1px stroke цветом `--ink`/`--pen`, без заливок (аксонометрии, схемы работы плагинов). SVG
- Осевые/штрихпунктирные линии (`stroke-dasharray`) для вспомогательной графики
- border-radius: 2px максимум. Сетка 12 колонок, max-width 1200px, щедрый воздух

## 6. Signature-элемент (hero главной)

Интерактивная **линейчатая поверхность**: две кривые Безье, между ними линейчатая поверхность из ~40 образующих линий. Пользователь таскает контрольные точки — поверхность перестраивается в реальном времени.

- three.js, LineSegments, цвет --ink, образующие --pen с прозрачностью; фон --paper, без сетки/осей
- Лёгкое ambient-вращение в покое; drag контрольных точек (raycast по сферам-ручкам)
- `prefers-reduced-motion` → статичный кадр (SVG fallback), также fallback для no-WebGL
- Рядом — заголовок и подзаголовок `[CONTENT]` + два CTA: "Browse plugins" / "Custom development"
- Бюджет: ≤ 150 KB JS на элемент, dynamic import, не блокирует LCP

Это единственное «смелое» место. Остальной сайт — тихий и дисциплинированный.

## 7. Моушн-правила

- Разрешено: fade/translate 200–300ms при входе секций в viewport (IntersectionObserver, один раз), hover-переходы 150ms, hero-элемент
- Запрещено: scroll-hijack, pin-секции, parallax, анимация текста по буквам, Lenis
- `prefers-reduced-motion: reduce` отключает всё, кроме hover

## 8. i18n

- next-intl, словари `en.json` / `ru.json`. Контент плагинов — поля `en`/`ru` в объекте контента
- hreflang, локализованные OG

## 9. SEO / OG

- Каждый `/plugins/[slug]` — title "Plugin Name — what it does | Nordlab", описание ≤ 155 зн. `[CONTENT]`
- OG-изображения на каждый плагин: кадр из демо + название, шаблонная генерация (`@vercel/og` или статика) `[CONTENT: кадры]`
- sitemap.xml, robots, JSON-LD `SoftwareApplication` на страницах плагинов

## 10. Перформанс и доступность

- Видео: H.264 mp4 ≤ 3–5 MB, постеры webp, `preload="none"` ниже фолда
- LCP < 2.0s на 4G, Lighthouse 90+ по всем категориям
- Видимый keyboard focus (outline --pen), контраст AA, alt на всю графику
